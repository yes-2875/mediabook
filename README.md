## CONTRIBUTOR GUIDE
If editing code, **DO NOT** fork the repository, use shared **dev/ branches** in this repository and push commits through the **Source Control tab in Visual Studio Code**.
There are **multiple** of these dev/ branches to avoid excessive commit conflicts, but you'll have to **pull from main to sync** them if they are outdated **BEFORE** working on the code.
Afterwards **create pull requests** (and issues to solve with the PRs if there are none) from your branch and the project maintainer will review before merging into main.
